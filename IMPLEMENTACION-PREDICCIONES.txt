🔧 ARQUITECTURA TÉCNICA
════════════════════════════════════════════════════════════════════════════════

FLUJO DE DATOS:
   GroupsPage
      ↓
   useMatches() → Array de Match[]
      ↓
   {matches.map(m => <MatchPredictionForm match={m} />)}
      ↓
   MatchPredictionForm
      ↓
   usePrediction() → PredictionContext
      ├─ savePrediction()
      ├─ getPrediction()
      ├─ canPredict()
      └─ isPredictionLocked()
      ↓
   localStorage["mockPredictions"]

ESTADOS DE UN PARTIDO:
   
   1. SIN PREDICCIÓN, ABIERTO (>24hs)
      ├─ Inputs habilitados
      ├─ Chip: "⚠️ X horas restantes"
      └─ Botón "Guardar Predicción"

   2. SIN PREDICCIÓN, CERRADO (<24hs)
      ├─ Inputs deshabilitados
      ├─ Chip: "⏰ Límite pasado"
      └─ Alert: "No se puede predecir"

   3. CON PREDICCIÓN
      ├─ Card mostrando predicción
      ├─ Timestamp: "Cargada hace X minutos"
      ├─ Chip: "✅ Predicción guardada"
      └─ No se puede cambiar

   4. PARTIDO FINALIZADO
      ├─ Muestra resultado oficial
      ├─ Compara con predicción
      ├─ Muestra: "✅ Exacto (+3 pts)" o "✅ Ganador (+1 pt)" o "❌ No acertaste"
      └─ (Sin calcular puntos aún - lógica pendiente)

⚙️ CÓMO FUNCIONA LA LÓGICA DE BLOQUEO
════════════════════════════════════════════════════════════════════════════════

Código en PredictionContext.tsx:

  canPredict(matchDate: Date): boolean {
    const hoursUntil = (matchDate.getTime() - Date.now()) / (1000 * 60 * 60);
    return hoursUntil > 24;
  }

  isPredictionLocked(matchDate: Date, matchId: string, userId: string): boolean {
    const key = `${userId}-${matchId}`;
    const hasPrediction = !!predictions[key];
    const isAfterDeadline = !canPredict(matchDate);
    
    return hasPrediction || isAfterDeadline;
  }

Interpretación:
  • isPredictionLocked = true si:
    1. El usuario ya tiene predicción, O
    2. Faltan menos de 24 horas para el partido

  • isPredictionLocked = false si:
    1. El usuario NO tiene predicción, Y
    2. Faltan más de 24 horas para el partido

⏱️ SIMULAR BLOQUEO DE 24 HORAS
════════════════════════════════════════════════════════════════════════════════

Para probar que después de 24 horas se bloquea:

1. Abre DevTools (F12)
2. Console tab
3. Ejecuta:

   // Cambiar fecha de un partido a 23 horas en el futuro
   const matches = JSON.parse(localStorage.getItem('mockMatches') || '[]');
   if (matches.length > 0) {
     matches[0].scheduledDate = new Date(Date.now() + 23 * 3600000);
     localStorage.setItem('mockMatches', JSON.stringify(matches));
   }
   location.reload();

Resultado:
   - El chip mostrará "⚠️ 23 horas restantes"
   - Los inputs estarán habilitados
   - Puedes hacer predicción

Después haz:
   // Cambiar a 12 horas
   matches[0].scheduledDate = new Date(Date.now() + 12 * 3600000);
   localStorage.setItem('mockMatches', JSON.stringify(matches));
   location.reload();

Resultado:
   - El chip mostrará "⚠️ 12 horas restantes"
   - Los inputs siguen habilitados
   - Puedes cambiar SOLO si no guardaste antes

Después:
   // Cambiar a -1 hora (pasado)
   matches[0].scheduledDate = new Date(Date.now() - 3600000);
   localStorage.setItem('mockMatches', JSON.stringify(matches));
   location.reload();

Resultado:
   - El chip mostrará "⏰ Límite pasado"
   - Los inputs estarán DESHABILITADOS
   - No puedes hacer predicción
   - Si tenías predicción, se sigue mostrando
