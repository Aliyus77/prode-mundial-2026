FLUJO DE EJECUCIÓN
════════════════════════════════════════════════════════════════════════

    1. Tu Computadora
       ├─ Terminal abierta en la carpeta
       └─ Node.js 18+ instalado

          ↓

    2. Ejecutar uno de estos:
       ├─ SETUP-COMPLETO.bat  (Windows automático)
       ├─ node create-all-files.js (genera archivos)
       └─ npm install (instala deps)

          ↓

    3. npm run dev
       ├─ Vite compila el código
       ├─ Inicia servidor local:5173
       └─ Abre navegador automáticamente

          ↓

    4. Navegador en http://localhost:5173
       ├─ Redirige a /login
       ├─ Login: admin@prode.com / admin123
       └─ Acceso a toda la aplicación

    │   └── Match cards

    │   └── Ranking display
    └── Route: /admin → AdminDashboard (ProtectedRoute + Admin only)
        ├── Tabs
        ├── UsersDashboard
        ├── PrizesForm
        ├── PrizeHistory
        ├── ImageUpload
        └── ResultsForm


FLUJO DE AUTENTICACIÓN
════════════════════════════════════════════════════════════════════════

Login Form
    ↓
useForm + Zod validation
    ↓
✓ Valid → useAuth.login()
    ↓
AuthContext → mockApi.login()
    ↓
✓ Credenciales correctas
    ↓
localStorage.setItem(token, user)
    ↓
State: {user, token}
    ↓
ProtectedRoute verifica user
    ↓
Renderiza página protegida
    ↓
En logout: limpiar localStorage → Redirige /login


FLUJO DE ADMINISTRACIÓN
════════════════════════════════════════════════════════════════════════

Admin Dashboard (Tab: Usuarios)
    ├─ Cargar usuarios: mockApi.getUsers()
    ├─ Filtrar por nombre/DNI
    └─ Mostrar tabla

Admin Dashboard (Tab: Premios)
    ├─ Formulario: nombre, descripción, criterio
    ├─ Validar con Zod
    ├─ mockApi.createPrize()
    └─ Notificación de éxito

Admin Dashboard (Tab: Historial)
    ├─ mockApi.getPrizeAssignments()
    └─ Mostrar tabla de histórico

Admin Dashboard (Tab: Banner)
    ├─ File input → Base64
    ├─ mockApi.uploadBanner()
    └─ Se muestra en GroupsPage

Admin Dashboard (Tab: Resultados)
    ├─ Cargar: homeScore, awayScore
    ├─ Verificar: hora límite
    ├─ mockApi.loadMatchResult()
    └─ Calcular automáticamente puntos


FLUJO DE PUNTUACIÓN
════════════════════════════════════════════════════════════════════════

Admin carga resultado
    ↓
mockApi.loadMatchResult()
    ↓
Compara: predicción vs resultado real
    ↓
    ├─ Exacto (ej 2-1 = 2-1) → 3 puntos
    ├─ Ganador correcto → 1 punto
    └─ Incorrecto → 0 puntos
    ↓
Se actualiza en estado
    ↓
Podium page muestra ranking nuevo
    ↓
Se pueden asignar premios


TECNOLOGÍAS POR CAPA
════════════════════════════════════════════════════════════════════════

UI LAYER (Lo que ve el usuario)
    ↓
    Material-UI (MUI)
    └─ Buttons, Forms, Tables, Cards, etc.

STATE LAYER (Lógica de aplicación)
    ↓
    Context API
    ├─ AuthContext (usuario, token, login, logout)
    └─ NotificationContext (mostrar toasts)
    
    React Hooks
    ├─ useState, useEffect, useContext
    └─ Custom: useAuth, useNotification, useMatches

FORM LAYER (Manejo de formularios)
    ↓
    React Hook Form
    └─ Gestión de inputs, validación

VALIDATION LAYER (Validar datos)
    ↓
    Zod
    └─ Schemas para login, register, premio, etc.

DATA LAYER (Datos)
    ↓
    Mock API (mockApi.ts)
    └─ Simula respuestas HTTP con setTimeout

DATA PERSISTENCE
════════════════════════════════════════════════════════════════════════

├─ users[] array
├─ matches[] array
├─ prizes[] array
├─ prizeAssignments[] array
└─ predictions[] array

├─ user (JSON serializado)
└─ Recupera al recargar página


MODELO DE DATOS
════════════════════════════════════════════════════════════════════════

User
├─ id, name, email, dni, password
├─ role: 'admin' | 'user'
├─ companyCode?: string
└─ createdAt

Match
├─ id, homeTeam, awayTeam, homeTeamFlag, awayTeamFlag
├─ group, scheduledDate, resultDeadline
├─ homeScore?, awayScore?
└─ isFinished

Prize
├─ id, name, description
├─ criteria: 'most_points_date' | 'most_points_phase' | 'most_points_tournament'
├─ assignmentType: 'automatic' | 'manual'
└─ createdAt

PrizeAssignment
├─ id, prizeId, userId, assignmentDate
├─ criteria, phase?
└─ [Historial de premios entregados]

Score
├─ userId, userName
├─ totalPoints, datePoints, phasePoints
└─ position
