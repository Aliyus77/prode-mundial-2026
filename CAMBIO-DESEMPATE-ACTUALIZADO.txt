CAMBIO IMPLEMENTADO:

El criterio "ENTREGAR AL PRIMERO" ahora premia al usuario que CARGÓ 
SUS RESULTADOS PRIMERO (no por ID ni fecha de registro).

════════════════════════════════════════════════════════════════════════════════
📊 COMPARACIÓN
════════════════════════════════════════════════════════════════════════════════

ANTES:
  ✗ "Gana el primero según ID o fecha de registro"
  ✗ Arbitrario
  ✗ No incentiva actitud

AHORA:
  ✅ "Gana el primero que cargó los resultados ganadores"
  ✅ Basado en timestamp de carga
  ✅ Premia rapidez y precisión
  ✅ Es rastreable y auditable


════════════════════════════════════════════════════════════════════════════════
⚙️ CÓMO FUNCIONA
════════════════════════════════════════════════════════════════════════════════

1. USUARIOS CARGAN RESULTADOS
   • Cada usuario que carga predicción → se registra timestamp
   • localStorage["mockPredictions"][userId].loadedAt = Date.now()

2. FINALIZA LA FASE/FECHA/TORNEO
   • Se calcula quién tiene más puntos
   • Si hay EMPATE en puntos:

3. DESEMPATE AUTOMÁTICO
   • Sistema busca a usuarios empatados
   • Verifica cuándo cargó cada uno
   • 🏆 GANA: quien cargó PRIMERO

4. PREMIO ASIGNADO
   • Se registra en historial
   • Notificación al ganador

mockPredictions (NUEVO):

  }

En caso de empate → gana userId_1 (cargó primero)


════════════════════════════════════════════════════════════════════════════════
🔧 FUNCIÓN IMPLEMENTADA
════════════════════════════════════════════════════════════════════════════════

En src/contexts/PrizeContext.tsx:

getFirstUserToLoadResults(userIds: string[]): string | null

PARÁMETROS:
  userIds: string[] → lista de usuarios empatados

RETORNA:
  string | null → userId del que cargó primero

LÓGICA:
  1. Lee mockPredictions de localStorage
  2. Para cada usuario, obtiene loadedAt
  3. Encuentra el timestamp mínimo
  4. Retorna ese userId
  5. Si no hay data, retorna primer ID (fallback)

USO:
  const winner = getFirstUserToLoadResults(["user1", "user2", "user3"]);
  // Retorna el usuario que cargó primero
