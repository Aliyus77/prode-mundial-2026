  ✅ RESOLVER CONFLICTO DE GIT PUSH
════════════════════════════════════════════════════════════════════════════════

PROBLEMA:
  ! [rejected] main -> main (fetch first)
  
  El repositorio remoto en GitHub tiene archivos que tu local no tiene
  (probablemente README.md o LICENSE creado automáticamente)

SOLUCIÓN: 2 OPCIONES

════════════════════════════════════════════════════════════════════════════════
OPCIÓN 1: SIMPLE (Recomendado)
════════════════════════════════════════════════════════════════════════════════

Ejecuta en terminal:

  git pull origin main

Esto:
  ✅ Trae los cambios remotos a tu local
  ✅ Los mergea automáticamente
  ✅ Luego puedes hacer push sin problemas

Después ejecuta:

  git push -u origin main

¡Listo! Tu código estará en GitHub.

════════════════════════════════════════════════════════════════════════════════
OPCIÓN 2: CON REBASE (Más limpio)
════════════════════════════════════════════════════════════════════════════════

Si prefieres un historial más limpio sin merge commits:

  git pull --rebase origin main

Luego:

  git push -u origin main

🆘 SI AÚN HAY PROBLEMAS
════════════════════════════════════════════════════════════════════════════════

Si hay conflictos después de git pull:

1. Git te mostrará los archivos en conflicto
2. Edita los archivos y resuelve los conflictos
3. git add .
4. git commit -m "Merge remote changes"
5. git push -u origin main

Si quieres hacer push fuerza (cuidado, sobrescribe):

  git push -u origin main --force

(Usar solo si estás seguro de qué haces)
